Product Requirements Document

StarMind：主动知识 Agent

StarMind 通过用户授权的数据源，主动同步用户在各平台的收藏、喜欢、星标、播放列表和稍后看内容， 自动完成内容抽取、摘要、标签、聚类、知识图谱关联和持续维护；同时保留“用户主动发送链接”的被动入口，用于覆盖微信、公众号等封闭平台。

1. 产品一句话定义

StarMind 是一个后台运行的主动知识 Agent。 用户授权后，它会持续观察用户在不同平台的收藏行为，将分散在抖音、小红书、B站、YouTube、X、知乎等平台里的内容， 自动转化为一个持续生长、自我维护的个人知识库。

核心定位

不是收藏夹导入器

不是一次性把链接搬进数据库，而是持续监听新增收藏和兴趣信号。

不是摘要工具

不是简单生成摘要，而是做去重、聚类、知识关联、复习和更新。

不是全平台爬虫

开放平台走官方/API/网页登录授权；封闭平台通过主动发送链接作为兜底。

2. 背景与问题

用户现状

  • 用户在多个平台收藏了大量内容：视频、图文、文章、帖子、代码项目、问答、PDF。
  • 收藏行为本身代表“未来可能有用”，但大多数收藏夹最终变成沉睡仓库。
  • 用户没有时间逐条整理，也没有动力持续维护知识库。
  • 封闭平台如微信收藏、公众号星标等缺少稳定的第三方自动读取路径。

核心问题

问题 用户痛点 StarMind 解决方式
收藏分散 信息散落在不同平台，无法统一检索和复用。 通过 Connector 主动同步多平台收藏信号。
收藏不等于知识 收藏后很少回看，无法沉淀为长期记忆。 自动抽取、摘要、标签、聚类、建立知识图谱。
维护成本高 知识库需要长期整理，用户难以坚持。 Maintenance Agent 周期性去重、更新、复习、清理。
封闭平台难接入 微信等平台不能稳定自动读取收藏。 保留被动入口：用户发送链接/文章给 Agent 后自动处理。

3. 产品目标与非目标

产品目标

  • 实现“授权一次，后台持续同步”的主动知识采集体验。
  • 支持主流社交媒体和内容平台的收藏、喜欢、星标、播放列表读取。
  • 将不同来源内容统一标准化为 Knowledge Item。
  • 通过 Agent 自动完成知识维护，而非仅做摘要。
  • 为无法主动读取的平台提供低摩擦被动入口。

非目标

  • 不承诺读取所有平台的私有收藏夹。
  • 不要求用户提供账号密码、Cookie 或 Session Token。
  • 不绕过验证码、登录限制、付费墙或平台权限。
  • 不把微信收藏作为第一阶段主动同步能力。
  • 不把“抓取数量”作为核心价值，核心价值是知识维护。

4. 核心用户与使用场景

用户类型 典型行为 需求 StarMind 价值
研究生 / 学习型用户 收藏论文、课程视频、公众号文章、知乎回答。 把零散资料变成可复习知识体系。 自动归类主题、生成复习卡片和知识图谱。

内容重度消费者 在小红书、抖音、B站、YouTube 高频收藏。 避免收藏夹沉睡，找回真正有用内容。 把娱乐式收藏筛选为长期知识资产。

5. 产品原则：主动为主，被动为辅

主动路径：主功能

用户勾选平台并完成授权后，StarMind 后台自动同步用户收藏、喜欢、星标、播放列表等内容。

用户勾选平台
→ 完成登录/授权
→ Connector 定时同步
→ Agent 判断知识价值
→ 自动入库与维护
核心竞争力 默认体验

被动路径：兜底功能

当某些平台无法主动读取收藏夹时，用户可以主动把链接/文章/图片/文件发送给 StarMind。

用户发送链接/文章
→ StarMind 识别平台
→ 抽取正文/逐字稿/OCR
→ 进入同一知识处理链路
覆盖封闭平台 例如微信公众号
产品表述建议： StarMind 不是“全平台爬虫”，而是“主动知识 Agent”。对于开放或网页可见的平台，StarMind 主动同步； 对于微信、公众号等封闭平台，StarMind 提供发送链接的低摩擦入口作为兜底。

6. 总体架构图

StarMind 由“数据源连接层、采集同步层、内容解析层、Agent 处理层、知识存储层、用户交互层”组成。

8. MVP 功能范围

8.1 主动采集 Connector

平台 采集对象 优先级 MVP 目标
Bilibili 收藏夹、稍后再看、视频元数据 P0 读取收藏视频列表，转为知识卡片，视频内容走字幕/ASR。
抖音 收藏视频、喜欢视频 P0 读取用户自己可见的收藏/喜欢列表，调用豆包或 ASR 获取逐字稿。
小红书 收藏笔记、点赞笔记 P0 读取笔记链接、标题、作者、封面；图文走 OCR/正文抽取。
知乎 收藏夹、回答、文章 P1 读取收藏内容，抽取正文与问答结构。
YouTube Liked videos、Playlists P0 读取视频列表，字幕优先，ASR 兜底。
X / Twitter Bookmarks、Likes P1 读取帖子和 thread，保留原文结构和外链。

8.2 被动输入能力

入口 覆盖场景 处理方式
链接输入框 用户粘贴任意文章/视频/PDF 链接。 URL Router 识别平台后进入 Extractor。
微信/飞书机器人 公众号文章、微信群链接、文件转发。 机器人接收消息，转发给 StarMind API。
文件上传 PDF、图片、截图、导出的书签文件。 PDF Parser / OCR / HTML Bookmark Parser。

8.3 知识处理能力

  • 标题、作者、平台、URL、收藏时间、封面等元数据提取。
  • 文章正文抽取、视频逐字稿、OCR、PDF 解析。
  • 一句话摘要、结构化摘要、关键词、标签、实体识别。
  • 自动去重：同一内容多平台出现时合并。
  • 主题聚类：把收藏内容归入自动生成的研究主题。
  • 知识图谱：内容之间建立“相似 / 解释 / 补充 / 冲突 / 延伸”关系。
  • 周期性维护：每日/每周生成知识更新报告。

9. 用户核心流程

流程 A：主动同步

1. 用户进入 StarMind Onboarding
2. 用户勾选想接入的平台：B站 / 抖音 / 小红书 / YouTube / X
3. 系统打开授权或网页登录流程
4. 用户完成登录并确认“允许 StarMind 读取自己可见的收藏列表”
5. Connector 拉取最近 N 条收藏内容
6. Filter Agent 过滤低价值/重复/娱乐噪音内容
7. Extractor 抽取正文、字幕、逐字稿或图片文字
8. Knowledge Agent 生成摘要、标签、实体和主题
9. Maintenance Agent 建立知识关联并生成首次知识图谱
10. 后台 Scheduler 持续同步新增收藏

流程 B：被动补充

1. 用户在微信看到公众号文章
2. 用户转发给 StarMind 机器人，或复制链接到 StarMind
3. StarMind 识别为微信公众号文章
4. 抽取正文、标题、作者、发布时间
5. 进入与主动路径完全相同的知识处理链路
6. 如果内容与已有主题相关，自动合并到对应主题

10. Agent 设计

Agent 职责 输入 输出
Observer / Collector Agent 监听和拉取用户授权数据源中的新增收藏。 平台连接状态、收藏列表、增量游标。 Raw Item 列表。
Filter Agent 判断内容是否值得进入知识库，过滤低价值噪音。 标题、平台、作者、内容类型、用户兴趣画像。 Knowledge Worthiness Score。
Extractor Agent 抽取正文、逐字稿、图片文字、PDF 文本。 URL、页面 HTML、视频链接、图片、PDF。 Clean Text / Transcript。
Knowledge Agent 生成摘要、标签、实体、主题和知识卡片。 Clean Text / Transcript。 Knowledge Item。
Maintenance Agent 持续维护知识库，做去重、合并、更新、复习和清理。 新旧 Knowledge Items、用户行为反馈。 知识图谱更新、复习报告、待处理建议。

12. 平台接入策略

平台类型 示例 接入方式 产品策略
Web 收藏页可见 B站、微博、知乎、小红书、抖音 网页登录 + 浏览器自动化 + 增量同步 主动路径优先支持。

13. MVP 页面设计

13.1 Onboarding：选择数据源

  • 标题：连接你的收藏源。
  • 副标题：选择你常用的平台，StarMind 会在后台自动同步新增收藏。
  • 平台卡片：B站、抖音、小红书、知乎、YouTube、X、GitHub。
  • 每个平台显示：可同步对象、同步方式、隐私说明、连接按钮。

13.2 Knowledge Inbox：待处理知识收件箱

  • 展示新同步但尚未完全处理的内容。
  • 显示处理状态：已抓取 / 正在解析 / 已入库 / 需要用户确认。
  • 支持用户手动保留、忽略、加入专题。

13.3 Knowledge Map：知识图谱

  • 以主题为中心展示内容节点。
  • 节点类型包括：视频、文章、图文笔记、问答、代码项目、PDF。
  • 边关系包括：相似、补充、引用、冲突、延伸。

13.4 Daily Reflection：每日知识报告

  • 今日新增内容数量。
  • 新增主题和高频关键词。
  • 值得复习的旧内容。
  • Agent 发现的知识缺口和后续阅读建议。

13.5 Passive Add：手动补充入口

  • 粘贴链接输入框。
  • 上传文件按钮。
  • 微信/飞书机器人入口说明。
  • 提示语：用于微信、公众号等无法主动同步的平台。

16. 风险与边界

风险 影响 应对策略
平台反自动化 Connector 失效或同步失败。 低频同步、人工登录、不绕过验证码、失败后降级为链接输入。
隐私风险 用户对收藏、历史、私密内容敏感。 明确授权范围；不保存账号密码；支持删除数据；敏感内容本地处理优先。
内容解析失败 无法生成摘要或逐字稿。 多 Extractor 兜底：字幕、ASR、OCR、人工链接补充。
知识库污染 娱乐/低价值内容过多。 Filter Agent 评分；用户可设置兴趣主题和忽略规则。
被质疑“只是爬虫” 产品价值被低估。 路演重点放在主动维护循环和知识图谱，而非抓取本身。

17. 黑客松 Demo 脚本

场景：一个用户平时在 B站、小红书、YouTube 收藏大量 AI Agent 和知识管理内容。

1. 用户打开 StarMind。
2. 勾选 B站、小红书、YouTube 三个平台。
3. StarMind 读取最近收藏内容。
4. 系统自动生成 20 张知识卡片。
5. Agent 识别出 3 个主题：
   - AI Agent
   - 个人知识管理
   - 自动化工作流
6. StarMind 自动发现：
   “你在 B站收藏的 Karpathy 视频”和“小红书的知识库笔记”属于同一个主题。
7. 系统生成知识图谱。
8. 用户再把一篇微信公众号文章发送给 StarMind。
9. 文章被自动归入“AI Agent”主题。
10. StarMind 输出今日知识报告：
    “你今天新增了 7 个 Agent 相关知识点，我已经合并为一个研究专题，并发现 2 个可继续学习的问题。”

18. 路演核心话术

StarMind is not a bookmark importer. It is a background knowledge agent.

StarMind 主动同步用户授权平台上的收藏、喜欢、星标和播放列表，把这些分散的弱收藏行为转化为结构化知识。 对于微信、公众号等无法主动同步的平台，StarMind 提供链接发送入口作为兜底。最终，用户不再需要手动维护知识库， 因为 StarMind 会持续帮用户收集、理解、连接、更新和复习知识。
