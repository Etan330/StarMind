Product Requirements Document / V0.2

StarMind：主动知识筛选与维护 Agent

StarMind 面向高频收藏但缺少整理能力的知识工作者。它不会把用户收藏夹里的所有内容都机械入库， 而是把收藏、喜欢、星标、播放列表等行为视为“候选信号”，由评分 Agent 判断是否值得沉淀为长期知识资产。 主动同步是核心能力；链接输入是封闭平台的兜底路径。

0. 本版核心修订

上一版问题 本版修订 原因
默认“收藏 = 知识” 改为“收藏 = 候选信号”，必须经过 Knowledge Worthiness Score 评分。 收藏夹里大量内容可能是娱乐、消费、临时兴趣、重复信息，直接入库会污染知识库。
产品定位偏功能描述 重写目标用户、核心痛点、愿景和差异化。 需要明确 StarMind 不是收藏导入器，而是主动筛选和维护知识资产的 Agent。
Agent 主动性表达不够具体 拆成主动采集、主动筛选、主动维护三层主动。 仅仅自动抓取不是壁垒，真正壁垒是判断什么值得保留，并持续维护。
被动入口与主动入口关系不清晰 明确被动入口只作为封闭平台兜底，所有内容进入同一评分和知识处理链路。 避免被动入口变成另一个普通“链接总结工具”。

1. 产品定位

StarMind 是一个主动知识筛选与维护 Agent。
它通过用户授权的数据源持续观察收藏行为，把分散在多平台里的“弱收藏信号”转化为候选内容； 再通过评分机制筛选出真正值得长期沉淀的内容，自动完成摘要、聚类、关联、复习和更新。

一句话定位

StarMind 把用户散落在各平台收藏夹里的潜在知识资产，筛选、组织并维护成一个持续生长的个人知识系统。

产品边界

不是收藏夹搬运工具

StarMind 不追求把所有收藏都塞进知识库，而是只保留值得长期复用的内容。

不是简单摘要工具

摘要只是中间产物，最终目标是形成主题、连接、复习和可检索知识资产。

不是全平台黑盒爬虫

主动同步依赖用户授权和网页可见内容；封闭平台走主动发送链接兜底。

愿景

让收藏夹从“信息坟场”变成“会自我筛选、组织和更新的第二大脑”。 StarMind 的长期愿景不是帮用户保存更多信息，而是减少无效信息沉积，让真正有价值的内容持续服务用户的学习、研究和决策。

2. 目标用户

StarMind 不应面向所有互联网用户。第一阶段应聚焦“高信息密度 + 高频收藏 + 有知识复用需求”的人群。

核心用户画像

用户类型 典型身份 收藏行为 核心痛点 StarMind 价值
学习研究型用户 研究生、考研/考证用户、独立研究者 收藏课程视频、论文解读、公众号文章、知乎回答、B站教程。 资料多但没有体系，复习时找不到重点。 把零散收藏整理成主题知识库和复习卡片。
行业观察型用户 产品经理、创业者、投资/咨询/商业分析从业者 收藏行业报告、竞品动态、融资新闻、专家观点、趋势分析。 信息输入多，但难以沉淀成可复用洞察。 自动聚合主题，形成行业观察和决策素材。
技术成长型用户 开发者、AI 工具玩家、自动化实践者 Star GitHub 项目，收藏技术文章、YouTube/B站教程、工具测评。 工具、框架、教程分散，难以形成完整学习路径。 连接项目、教程、概念和实践案例。
创作者型用户 内容创作者、知识博主、课程制作者 收藏选题、案例、观点、素材、数据和同行内容。 素材沉睡在收藏夹里，产出时难以快速调取。 把收藏转成选题库、案例库、观点库。

非目标用户

第一阶段不建议服务纯娱乐收藏用户。 如果用户主要收藏搞笑视频、购物种草、明星动态、临时消费内容，StarMind 的知识库价值不明显。 这些内容可以被同步，但默认不应进入核心知识库。

3. 用户核心问题

01
收藏太容易，整理太难

用户频繁收藏，但不会定期整理，收藏夹逐渐变成不可用仓库。

02
平台太分散

同一主题内容分布在 B站、抖音、小红书、知乎、YouTube、X 等平台。

03
价值混杂

收藏夹里同时存在知识、娱乐、消费、临时兴趣、重复内容，直接入库会污染系统。

04
知识不会自我更新

即使用户手动建库，后续也需要持续打标签、去重、复习、更新，维护成本高。

StarMind 要解决的核心问题

用户不是缺少收藏工具，而是缺少一个能判断“什么值得成为知识”、并持续维护这些知识的主动 Agent。

4. 产品解法：三层主动 + 一层兜底

第一层主动：主动采集

用户勾选平台并完成授权后，StarMind 定时同步用户自己可见的收藏、喜欢、星标、播放列表等内容。

B站 抖音 小红书 YouTube X

第二层主动：主动筛选

同步到的内容不会直接进入知识库，而是进入 Candidate Pool，由评分 Agent 判断是否值得沉淀。

Knowledge Worthiness Score Noise Filter

第三层主动：主动维护

进入知识库后，Agent 持续做去重、聚类、关联、复习、过期检测、知识缺口提示。

Knowledge Graph Maintenance Loop

兜底路径：被动输入

对微信、公众号等无法主动读取的平台，用户可以主动发送链接、文章、图片或文件，进入同一评分和处理链路。

微信 公众号 PDF / 图片

5. 核心机制：Knowledge Worthiness Score

StarMind 需要一个明确的评分机制，判断某条收藏是否应该进入知识库。该机制分为两阶段： 采集后初筛和内容解析后精筛。

5.1 评分公式

KWS = 
  0.20 × User Intent Signal        用户意图强度
+ 0.20 × Topic Relevance           与用户长期兴趣/当前项目的相关性
+ 0.20 × Information Density       信息密度
+ 0.15 × Novelty                   相对已有知识的新颖性
+ 0.10 × Actionability             是否能转化为行动/方法/案例
+ 0.10 × Source Quality            来源质量
+ 0.05 × Evergreen Value           长期价值
- Penalty                          噪音、重复、娱乐、低完整度、广告等惩罚项

5.2 分数解释

分数区间 处理策略 产品表现 示例
80-100 自动进入正式知识库 生成完整知识卡片、标签、实体、主题关联、复习问题。 用户长期关注 AI Agent，收藏了一篇高质量 Agent 架构文章。
60-79 生成轻量卡片或进入待确认 进入 Knowledge Inbox，用户可一键确认入库。 一条不错的视频教程，但与用户主线兴趣关联不强。
40-59 只归档，不进入核心知识库 可搜索，但不参与知识图谱和复习计划。 临时收藏的工具测评、消费参考、娱乐向内容。
0-39 默认忽略 不处理或仅保留原始链接日志。 重复内容、广告、纯娱乐短视频、抽奖活动、商品页面。

5.3 评分维度说明

维度 高分特征 低分特征
User Intent Signal 用户主动收藏、加入专辑、多次收藏同主题、长时间观看/阅读。 偶然点赞、短暂停留、误触收藏、一次性娱乐内容。
Topic Relevance 匹配用户设置的目标：AI、自动化、商业分析、学习专题等。 与用户长期主题无关，且没有明确项目关联。
Information Density 包含方法论、概念解释、案例、数据、结构化观点。 标题党、情绪表达、低信息量口播、纯展示内容。
Novelty 补充已有知识缺口，或提供新框架、新案例、新数据。 与已有知识高度重复，或只是同一观点重复表达。
Actionability 可转化为任务、学习路径、工作流、决策依据、案例库。 只能看一眼，没有后续复用价值。
Source Quality 可靠作者、权威媒体、技术文档、高质量社区、长内容。 营销号、搬运号、无来源、广告、低质量合集。
Evergreen Value 长期有效的方法、模型、概念、案例、基础知识。 短期热点、临时折扣、时效很短的娱乐新闻。

6. 总体产品架构图

相比上一版，本版增加了 Candidate Pool 和 Scoring Agent。所有主动/被动输入都必须经过评分，不直接进入知识库。

7. 关键功能需求

7.1 主动采集功能

功能 说明 优先级
平台勾选 用户选择需要同步的平台，如 B站、抖音、小红书、YouTube、X。 P0
网页登录授权 用户在本地或安全浏览器会话中完成登录，StarMind 仅读取用户可见收藏页。 P0
增量同步 只读取新增收藏或变更内容，避免重复处理。 P0
失败降级 若平台同步失败，提示用户通过链接输入补充。 P1

7.2 评分与筛选功能

功能 说明 优先级
Candidate Pool 所有采集内容先进入候选池，不直接进入知识库。 P0
Initial Score 基于平台、收藏动作、标题、作者、内容类型、用户兴趣做初筛。 P0
Final KWS 内容解析后，基于信息密度、相关性、新颖性和长期价值评分。 P0
用户反馈修正 用户可标记“这个有用/这个无用”，系统更新偏好。 P1

7.3 被动输入功能

入口 目标场景 要求
链接输入框 微信、公众号、临时网页、无法同步平台。 粘贴链接后自动识别平台并进入 Candidate Pool。
微信/飞书机器人 用户从微信/飞书转发文章、链接、图片、文件。 机器人收到后调用 StarMind API，所有内容进入评分链路。
文件上传 PDF、截图、图片笔记、书签导出文件。 PDF/OCR/HTML Parser 处理后进入评分链路。

7.4 知识维护功能

  • 自动去重：同一内容多平台出现时合并。
  • 主题聚类：根据语义自动生成主题，如“AI Agent 商业化”“飞书自动化”“机器人行业”。
  • 知识图谱：建立“解释、补充、相似、冲突、延伸”关系。
  • 复习机制：对高 KWS 内容生成复习问题和周期回顾。
  • 知识过期检测：对强时效内容标记过期风险。
  • 知识缺口提示：当用户持续关注一个主题但缺少基础概念时，主动提示补充。

9. 用户流程

流程 A：主动同步 + 自动筛选

1. 用户打开 StarMind，选择关注方向：
   - AI / 自动化 / 商业分析 / 学习 / 编程 / 创作素材

2. 用户勾选平台：
   - B站、抖音、小红书、YouTube

3. 用户完成登录或授权。

4. StarMind 拉取最近收藏内容，进入 Candidate Pool。

5. Scoring Agent 做初筛：
   - 判断平台、标题、作者、内容类型、用户意图。

6. 对候选内容做解析：
   - 视频转逐字稿，图文做 OCR，文章抽正文。

7. Scoring Agent 做精筛：
   - 输出 KWS 分数。

8. 系统按分数处理：
   - ≥80：进入核心知识库
   - 60-79：进入待确认
   - 40-59：只归档
   - <40：忽略

9. Maintenance Agent 对入库内容做摘要、标签、聚类、知识图谱和复习计划。

流程 B：封闭平台被动输入

1. 用户在微信看到公众号文章。
2. 用户把文章链接发给 StarMind Agent。
3. StarMind 不直接入库，而是先进入 Candidate Pool。
4. 文章正文被抽取后，Scoring Agent 计算 KWS。
5. 只有达到阈值的内容才进入知识库。
6. 如果该文章与已有主题相关，自动连接到已有知识图谱。

10. 用户偏好与评分冷启动

12. 页面与交互设计

12.1 Onboarding：定义我的知识目标

  • 用户选择长期关注方向：AI、商业、自动化、编程、学习、创作、投资、行业研究等。
  • 用户填写当前项目：例如“黑客松 StarMind”“AI Agent 商业化研究”。
  • 用户选择默认过滤内容：娱乐、购物、明星、抽奖、低质营销等。

12.2 Connector 页面：连接数据源

  • 平台卡片显示：可同步对象、风险提示、上次同步时间、待处理数量。
  • 每个平台允许设置权重：高、中、低。
  • 允许关闭某个平台的自动入库，仅做候选收集。

12.3 Candidate Pool 页面：候选内容池

  • 展示所有新采集内容和初筛分数。
  • 支持按平台、分数、主题、内容类型筛选。
  • 用户可批量确认、忽略、调整评分。

12.4 Knowledge Inbox 页面：待确认知识

  • 展示 60-79 分内容。
  • 让用户一键选择：入库、归档、忽略、永远忽略类似内容。
  • 用户反馈用于训练个人评分偏好。

12.5 Knowledge Map 页面：核心知识库

  • 仅展示 KWS ≥ 80 或用户确认入库的内容。
  • 按主题、标签、实体、时间线组织。
  • 支持图谱关系：相似、补充、解释、冲突、延伸。

13.2 Scoring Agent Prompt 方向

你是 StarMind 的知识价值评分 Agent。
你的任务不是总结内容，而是判断这条内容是否值得进入用户的长期知识库。

请基于以下信息评分：
1. 用户长期关注方向
2. 用户当前项目
3. 内容标题、作者、平台
4. 内容正文/逐字稿
5. 已有知识库主题
6. 用户明确排除的噪音类型

输出：
- 0-100 的 KWS 分数
- 各维度子分
- 是否入库的决策
- 简短理由
- 若不入库，应归档还是忽略

14. 成功指标

指标类型 指标 MVP 目标
主动性 主动采集内容占比 ≥ 70%。
筛选质量 用户确认“有用”的自动入库内容比例 ≥ 75%。
污染控制 核心知识库中低价值内容占比 ≤ 15%。
处理效率 从收藏同步到初筛完成 Demo 场景 ≤ 30 秒。
知识组织 自动聚类准确率 用户主观认可 ≥ 70%。
留存价值 用户每周查看知识报告次数 ≥ 2 次。

15. 风险与应对

风险 说明 应对策略
知识库污染 收藏夹中娱乐/消费/重复内容过多。 KWS 评分 + 阈值入库 + 用户反馈修正。
评分不准 冷启动时系统不了解用户偏好。 Onboarding 收集兴趣；待确认区获取反馈；逐步个性化。
平台同步失败 网页登录状态失效、页面结构变化、反自动化。 低频同步、手动重连、失败降级为链接输入。
用户隐私担忧 收藏夹可能包含私密兴趣或敏感内容。 明确授权范围、支持删除、仅处理用户选择平台、不读取账号密码。
产品被理解成爬虫 评委或用户可能只看到采集功能。 路演强调 Scoring Agent 和 Maintenance Agent，核心是筛选与维护。

17. 路演核心话术

StarMind is not a bookmark importer. It is an active knowledge filtering and maintenance agent.

用户的收藏夹里并不全是知识。它混杂了娱乐、消费、临时兴趣和重复信息。 StarMind 的核心能力不是“同步更多收藏”，而是主动判断什么值得成为长期知识资产。 它会把多平台收藏先放入候选池，通过 Knowledge Worthiness Score 评分筛选，再将高价值内容沉淀为知识卡片、主题和知识图谱。 对于微信和公众号等封闭平台，用户可以主动发送链接，仍然进入同一套评分和维护链路。
