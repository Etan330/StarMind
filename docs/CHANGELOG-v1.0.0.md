# Release v1.0.0 — 智能知识管理全流程

## 概述

StarMind v1.0.0 是一个重大版本更新，从"收藏夹同步工具"升级为**完整的智能知识管理系统**。新增了自动沉淀、分类管理、个性化推送、偏好调节等核心功能，并修复了多个关键 Bug。

---

## 新增功能

### 🧪 自动沉淀：RawSource → 知识库 (Feature 1)
- 新增 `AutoDistillService`，定时（每 30 分钟）扫描未沉淀的原始资料
- AI 深度提炼：不再照搬原文，而是生成结构化知识卡片（一句话总结 + 核心观点 + 可复用结论 + 适用场景 + 局限）
- 沉淀进度弹窗：实时显示进度条、AI 处理日志、每条完成状态
- 手动触发按钮「⚗️ 沉淀到知识库」

### 📁 知识库分类文件夹 (Feature 2)
- 新增 `WikiCategory` 模型，AI 自动分类（优先匹配现有分类，尽量少创建新类别）
- 知识库左侧导航改为动态分类文件夹视图
- 分类重命名：hover 显示 ✏️ 按钮，支持 inline 修改
- 偏好设置入口：分类面板顶部 ⚙️ 按钮打开滑块弹窗

### 🔔 Push 推送系统 (Feature 3)
- 用户可设置推送日期（周一至周日复选）+ 多个推送时间（可添加/删除时间槽）
- 按分类偏好权重随机选取知识推送（偏好高 = 出现概率高）
- 推送内容：标题 + 一句话总结摘要（不是整篇文章）
- 7 天内不重复推送同一内容
- 浏览器 Notification 桌面通知
- 定时器每分钟检查是否到达推送时间

### 👍👎 偏好自动调节 (Feature 4)
- 每 5 次推送出现 Like/Unlike 反馈按钮
- Like → 对应分类偏好 +10
- Unlike → 对应分类偏好 -10 + 该条知识移入「知识黑名单」分类
- 知识黑名单偏好永远为 0，后续推送自动排除

### ⏰ 连接器定时同步设置 (Feature 5)
- 同步收藏夹页面每个平台新增定时设置按钮
- 弹窗选择日期 + 时间，支持「应用到所有平台」
- 按 platform 正确关联 Connector（修复之前硬编码 id=1 的 Bug）
- 每个连接器独立 APScheduler 任务

### 🎨 滚轮时间选择器
- 全局替换所有 `input[type="time"]` 为滚轮式选择器
- 默认显示当前时间
- 小时/分钟滚轮无限循环

## Bug 修复

### 🔴 百度内部 API 连接失败 (Feature 0)
- **根因**：`providers.json` 的 `base_url` 为空字符串；`save_provider_base_url()` 缺少空字符串防护
- **修复**：增加空字符串防护 + `chat()` 方法增加 base_url 检查 + 恢复默认 URL

### 🔴 小红书点点提取失败
- **根因**：Diandian 的 React 页面不响应 `sendButton.click()`
- **修复**：改用 `Enter` 键盘事件发送消息
- 移除冗余的 `click_at()` 双击调用
- 确认循环从 6 次增加到 12 次
- 新增"输入框被清空 = 发送成功"的判断

### 🔴 LLM 沉淀超时
- **根因**：httpx 超时设置 30 秒太短，百度内部 API 响应慢时失败被静默吞掉
- **修复**：超时增加到 120 秒 + 增加异常日志

### 🟡 收藏夹抓取去重
- 扫描结果对比 SyncLedger + RawSource 已有 URL，过滤重复项
- 初始抓取量 = max(用户请求 × 3, 50)，确保滚过顶部重复区
- 全部重复时返回友好提示"收藏夹内容均已入库"

### 🟡 博主蒸馏去重
- `/distill/profile` 端点增加 `profile_url` 去重，相同博主不会重复添加
- 点点提取增加 `xiaohongshu_diandian_extracted` 检查，已提取不重复发送

### 🟡 弹窗无法关闭
- 修复 `hidden` 属性被 `display: flex` CSS 覆盖的问题
- 统一添加 `.modal-overlay[hidden] { display: none; }`
